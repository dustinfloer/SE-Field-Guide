Workflow

Keep the prompt. Add the control surface.

The seller experience stays simple: ask for a deck, review the plan, tune the story, preview locally, and export safely. Studio is the local workspace that makes the choices visible.

From first prompt to merchant follow-up
Start in natural language
Ask for a deck by merchant name. The skill finds local context, Salesforce notes, meeting artifacts, and known deal signals.
Plan the deck before polish
Studio classifies the deal, reads the deck manifest, and turns the plan into stable slide choices with in-deck, hidden, and addable states.
Choose slides, edit copy, and preview live
SEs and AEs can add registered slide options, remove slides that do not fit, edit supported fields, and preview the deck without touching raw HTML.
Share with the right artifact
Preview locally, publish a Quick-ready folder for internal/demo collaboration, save a reviewed Field Guide example when useful, and export PDF for merchant-safe follow-up.

Install

Run one installer, then launch Studio locally.

Demo Deck Studio runs on your machine from the SE Assistant repo. You do not need a local Field Guide folder; the installer can pull the latest released skill from GitHub, copy it into your SE Assistant workspace, and install the Studio app.

S
Local Studio v2
React/Vite frontend + local Node API
1Install or update the skill
curl -fsSL https://raw.githubusercontent.com/dustinfloer/SE-Field-Guide/main/tools/demo-deck-builder/install.sh | bash
Use this if you do not have the Field Guide repo, or if you just want the simplest update path. If your SE Assistant folder is somewhere custom, the installer will tell you how to pass the path.
cd /path/to/SE-Field-Guide git pull origin main bash tools/demo-deck-builder/install.sh --se-assistant /path/to/SE-Assistant
If you already have the Field Guide repo locally, this path keeps your local clone current before installing.
2Run Studio v2 from the SE Assistant repo
node .claude/skills/demo-deck-builder/studio/demo-deck-studio.mjs studio-v2 merchants/[merchant]/index.html --port 7332 --api-port 7333
3Publish or export when the deck is ready
node .claude/skills/demo-deck-builder/studio/demo-deck-studio.mjs publish merchants/[merchant]/index.html
node .claude/skills/demo-deck-builder/studio/demo-deck-studio.mjs publish merchants/[merchant]/index.html --field-guide-copy --field-guide-dir /path/to/SE-Field-Guide
Use Field Guide copy only after the deck is reviewed, polished, and safe as internal team inspiration.
node .claude/skills/demo-deck-builder/studio/demo-deck-studio.mjs export-pdf merchants/[merchant]/index.html
Keep the terminal running. Studio v2 opens http://127.0.0.1:7332/ automatically so you can choose slides, edit supported fields, Preview Deck locally, publish a Quick-ready folder, or export PDF. If the browser says 127.0.0.1 refused to connect, restart the Studio command.

Share / export

Use the right artifact for the right audience.

Studio does not replace the static deck. It gives SEs and AEs a safer way to shape it before sharing.

01
Preview Deck
Best for working review, rehearsal, and live demo control. This is the fastest way to iterate with the SE/AE team before publishing.
02
Quick-ready publish
Publishes the selected Studio deck to exports/quick/index.html for internal Shopify collaboration and demo sharing.
03
Field Guide copy
Optionally saves the selected deck to a local Field Guide examples folder when it is reviewed and useful for team inspiration.
04
PDF export
Best for merchant-safe external follow-up, including approved post-demo updates from call notes.
